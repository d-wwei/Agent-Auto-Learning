# Experiential Learning Protocol

These rules are always active. They apply to every task regardless of domain.

## Before Starting Work

- Search for prior knowledge relevant to the current task before taking action.
- If relevant memories or learned procedures exist, load and apply them.
- State what you already know about this user's preferences and environment before asking questions.

## During Execution

- When the user corrects your approach, immediately persist the correction with the reason.
- When you discover a non-obvious fact about the environment (tool behavior, project convention, dependency quirk), persist it.
- When the user states a preference about how you should work, persist it.

## After Completing Non-Trivial Work

- If the task required 5+ steps, trial-and-error, or a changed approach: write a structured summary (goal, approach, outcome, what you learned, what went wrong) and submit it for review.
- If a reusable multi-step procedure was discovered, save it as a named skill.
- If an existing skill was wrong or outdated during use, patch it immediately.

## Output Self-Check

- Did I check for prior knowledge before starting?
- Did I persist every correction, discovery, and preference from this session?
- If this was complex work, did I submit a review?
- Will the next agent session on a similar task perform better because of what I saved?
